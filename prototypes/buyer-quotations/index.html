Received Quotes

Every quotation vendors have submitted across your organization's RFQs, in one place.

Disclosed quotations only · states are the frozen quotation lifecycle · nothing here is ranked

Review note — what this page proposes. A buyer-org-wide quotation list does not exist in the frozen corpus (quotations are readable only per-RFQ via list_quotations_for_rfq or per-document via get_quotation). This page is the design artifact for ESC-BUY-QUOTES-LIST — the proposed additive read rfq.list_quotations_for_buyer_org.v1, an org-scoped aggregate of already-disclosed quotations. It adds no new disclosure: every row is data the buyer can already see on some P-BUY-09 page. The four tiles are the read's server-computed stateCounts summary (counts by frozen state + validity window) — contract projections, not client analytics (R7). Row links land on the already-built per-RFQ pages (P-BUY-14 detail · P-BUY-15 comparison). Use the Populated / Loading / Empty / Error switch (top-right) to review every list state.

Compare Quotes

Pick an RFQ to lay its quotations side by side. Comparison is always within one RFQ — quotes answering different requests aren't comparable.

Descriptive · not ranked

Review note — the design decision. A free "pick any quotes across any RFQs" comparison is deliberately not offered: the frozen comparison read (get_comparison_statement) is scoped to one RFQ, and mixing RFQs would invite exactly the cross-request ranking that R6 (comparison never recommends) forbids. This page is therefore an accelerator: it surfaces every RFQ that has two or more disclosed quotations and opens its existing per-RFQ comparison in place — one click instead of four. Vendors are shown in the order provided, never ranked; no best-value badge, no highlighted winner, no lowest-price emphasis.
