## Ratified decisions (FR1–FR12)
*(`FRn` = Doc-7F realization decisions — distinct from `DR-7-*`.)*

- **FR1 — Scope: the Buyer Workspace (moat).** Authenticated buyer end-to-end: discovery → RFQ authoring + internal approval → routing/invitation observability → quotation viewing → comparison → award → post-award operations → buyer-private CRM. Realizes the Buyer-leg of Doc-5E + Doc-5F + Doc-5D. Mounts in `(app)`, org-scoped (Buyer/Hybrid); consumes 7B/7C by reference.
- **FR2 — RFQ authoring & internal approval (Doc-5E §4 Buyer).** `create_rfq`/`update_rfq`/`submit_rfq`/`approve_rfq`/`reject_internal_rfq`/`cancel_rfq`/`reissue_rfq` + reads (`get_rfq`/`list_rfqs`/`get_rfq_version`). Internal approval mirrors factory spend authorization; RFQ state machine per Doc-4M.
- **FR3 — Routing & invitation observability (Doc-5E §7 caller reads; engine out-of-wire).** Bind the verified **caller legs** `get_routing_log` / `get_invitation` / `list_invitations`. **`get_matching_results` is NOT bound** (Admin leg + internal-service — anti-gaming; R5). **The matching/routing engine is System out-of-wire (§8); the buyer NEVER invokes matching/routing.** R5: explainability is non-disclosing; R7: no payment influences the display.
- **FR4 — Quotation viewing (Doc-5E §5; visibility-gated).** `get_quotation` / `list_quotations_for_rfq` (Buyer leg, **`quotation_visibility`-gated**). Vendor quotation authoring is Doc-7G.
- **FR5 — Comparison & Award (Doc-5E §6; R6 — the no-auto-decision moat).** `get_comparison_statement` (Buyer read — read-only decision support, generated by the System engine §8), `shortlist_quotation`, `manage_clarification` (+ M6 thread), `invoke_best_and_final`, **`award_rfq`** (explicit Buyer command), `close_lost_rfq`. **The UI — and the AI advisory panel — NEVER auto-recommend, auto-rank-to-winner, auto-select, or present a "recommended winner"** (`Doc-5E R6`; `Doc-3 §9.1` FIXED; Invariant #12). Award is a deliberate, unranked buyer choice; the comparison statement is decision support, not a recommendation.
- **FR6 — Post-award operations (Doc-5F buyer-leg; money-boundary R8).** Bind the buyer-leg post-award documents — those the buyer **issues** (PO, payment record) and **receives/approves** (challan, invoice, WCC) — + finance records; the **vendor-leg is Doc-7G**. Bound per Doc-5F actor leg at content (bind-or-ESC). **Platform handles no buyer↔vendor money** (R8); post-award docs are records/workflow, never settlement/escrow; a trade invoice is a record (≠ platform billing invoice — Doc-5I R6).
- **FR7 — Buyer-private CRM (Doc-5F BC-OPS-1; Invariant #11; `CHK-7-040`).** The buyer's private vendor records (status/notes/approved/blacklist) render **ONLY in the buyer's own workspace; never leak to any vendor-facing surface, view, count, notification, or analytic**. **Buyer Approved/Blacklisted never mutates platform-wide scores** (firewall; M4 CRM). The blacklist is undetectable by the vendor (byte-equivalence attestation is Doc-7G's).
- **FR8 — Discovery (Doc-5D buyer).** `search_catalog` / `list_vendor_directory` (vendor discovery to invite) + **favorites** (`add_catalog_favorite`/`remove_catalog_favorite`/`list_catalog_favorites` — BC-MKT-7, User membership-only).
- **FR9 — State-machine UI (Doc-4M).** RFQ/quotation/award/post-award document lifecycles render only Doc-4M-permitted, Buyer-actor transitions; **System engine transitions (matching/routing/expire/comparison-generation) displayed, never invoked**; STATE/CONFLICT (409) → reconcile.
- **FR10 — Firewall & non-disclosure (R5/R6/R7; Invariant #11).** R7 firewall — payment/entitlement never a wire input to matching/routing/selection nor influences their display; **billing quota = delivery ceiling** (three-instrument identity `Doc-3 §4.1.1`), **never a matching-eligibility gate** and never presents a vendor as excluded-by-quota. R6 — no auto-decision (FR5). R5 — non-disclosing explainability. Buyer-private CRM never leaks (FR7). Reads own-org / `quotation_visibility`-gated; cross-tenant/protected → `NOT_FOUND`.
- **FR11 — Composition & applicable Appendix A (full surface).** Composes Doc-7B embedded components: **trust badge** (`Doc-5G`), **billing/quota indicator** (`Doc-5I` — delivery ceiling only, R7), **AI advisory panel** (`Doc-5K` — advisory only, **never a winner recommendation**, R6/R10), **conversation thread** (`Doc-5H` — RFQ/clarification, non-disclosure-bound). Full `CHK-7-xxx` set APPLIES (richest surface); N/A reasons at content.
- **FR12 — Coins nothing.** Binds frozen Buyer-leg by pointer; never invokes the engine; gaps → flag-and-halt (`[ESC-7-API]`), never invent.
