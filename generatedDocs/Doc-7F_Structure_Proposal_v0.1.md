## Decisions proposed for ratification (R-set `FR1…FR12`)
*(`FRn` = Doc-7F realization decisions — distinct from `DR-7-*`.)*

- **FR1 — Scope: the Buyer Workspace (moat).** The authenticated buyer's end-to-end flow — discovery, RFQ authoring + internal approval, routing/invitation observability, quotation viewing, comparison, award, post-award operations, buyer-private CRM. Realizes the **Buyer-leg** of Doc-5E + Doc-5F + Doc-5D buyer discovery. Mounts in Doc-7C `(app)`, **org-scoped** (active org; Buyer/Hybrid participation); consumes the 7B kit + 7C server-side wired client by reference; authors its own views only.
- **FR2 — RFQ authoring & internal approval (Doc-5E §4 Buyer).** Bind `create_rfq`, `update_rfq`, `submit_rfq`, `approve_rfq`, `reject_internal_rfq`, `cancel_rfq`, `reissue_rfq` (Buyer User commands) + reads (`get_rfq`, `list_rfqs`, `get_rfq_version`). The **internal approval workflow** mirrors factory spend authorization (Master Architecture); the RFQ state machine renders per Doc-4M (FR9).
- **FR3 — Routing & invitation observability (Doc-5E §7 caller reads; engine out-of-wire).** Bind the buyer **observability reads**: `get_invitation`, `list_invitations`, `get_routing_log`, `get_matching_results` (Buyer/caller legs). **The matching/routing engine is System out-of-wire (§8) — the buyer NEVER invokes matching/routing** (`run_matching_pipeline`, `assemble_and_route_wave`, etc. are the engine). **R5: routing-log/matching reads must not expose a protected fact** (explainability is non-disclosing). **R7: no payment/entitlement value influences the matching/routing display.**
- **FR4 — Quotation viewing (Doc-5E §5 reads; visibility-gated).** Bind `get_quotation`, `list_quotations_for_rfq` (Buyer leg, **`quotation_visibility`-gated**). The buyer views received quotations; **vendor quotation authoring is Doc-7G** (not here).
- **FR5 — Comparison & Award (Doc-5E §6; R6 — the no-auto-decision moat).** Bind `get_comparison_statement` (Buyer read — **read-only decision support**, generated by the System engine `generate_comparison_statement` §8, never by the buyer), `shortlist_quotation`, `manage_clarification`, `invoke_best_and_final`, **`award_rfq`** (explicit Buyer command), `close_lost_rfq`. **The UI NEVER auto-recommends, auto-ranks-to-winner, auto-selects, or shows a "recommended winner"** (`Doc-5E R6`; `Doc-3 §9.1` FIXED) — **award is a deliberate buyer action**; the comparison statement is decision support, not a recommendation.
- **FR6 — Post-award operations (Doc-5F buyer-leg; money-boundary R8).** Bind the buyer-leg post-award documents (LOI/PO/challan/invoice/payment/WCC) + finance records (Doc-5F). **The platform handles no buyer↔vendor money** (`Doc-5F R8`) — post-award docs are **records/workflow, never settlement/escrow/custody**; a **trade invoice is a record (≠ platform billing invoice** — Doc-5I R6 boundary). (Exact Doc-5F buyer contracts bound at content; bind-or-ESC.)
- **FR7 — Buyer-private CRM (Doc-5F BC-OPS-1; Invariant #11; `CHK-7-040`).** Bind the buyer's **private vendor records** (status/notes/approved/blacklist). **Renders ONLY in the buyer's own workspace; never leaks to any vendor-facing surface, view, count, notification, or analytic** (Invariant #11). **Buyer Approved/Blacklisted never mutates platform-wide scores** (governance firewall; M4 CRM holds private per-buyer status). The blacklist is **undetectable** by the vendor (the byte-equivalence attestation is Doc-7G's; here, the CRM is private to the buyer).
- **FR8 — Discovery (Doc-5D buyer).** Bind `search_catalog`, `list_vendor_directory` (vendor discovery to invite) + **favorites** (`add_catalog_favorite`/`remove_catalog_favorite`/`list_catalog_favorites` — BC-MKT-7, **User membership-only**). The buyer discovers vendors to invite to RFQs.
- **FR9 — State-machine UI (Doc-4M).** RFQ lifecycle, quotation, award/closure, and post-award document lifecycles render **only Doc-4M-permitted, Buyer-actor transitions** (`Doc-7A §7`); **System engine transitions (matching/routing/expire/comparison-generation) are displayed, never invoked** (out-of-wire); STATE/CONFLICT (409) → reconcile.
- **FR10 — Firewall & non-disclosure (R5/R6/R7; Invariant #11).** **R7 firewall:** no payment/entitlement/plan value is a wire input to matching/routing/selection or influences their display; **billing quota is read as a delivery ceiling** (three-instrument accounting — `Doc-3 §4.1.1`), never a procurement gate the UI applies. **R6:** no auto-decision (FR5). **R5:** routing/matching explainability is non-disclosing. **Buyer-private CRM never leaks** (FR7). Reads are **own-org / `quotation_visibility`-gated**; cross-tenant/protected → `NOT_FOUND` (`Doc-5A §6.3`).
- **FR11 — Composition & applicable Appendix A (full surface).** Composes Doc-7B embedded components: **trust badge** (`Doc-5G` on vendors/quotations), **billing/quota indicator** (`Doc-5I` delivery-ceiling read), **AI advisory panel** (`Doc-5K` buyer reads — advisory only, non-gating, never an auto-decision — R6/R10), **conversation thread** (`Doc-5H` RFQ thread). Applicable `CHK-7-xxx`: **full set** APPLIES (richest surface — contract-binding/active-org/Content≠Presentation/state-machine/non-disclosure/AI/baseline/out-of-frontend/realize-never-redecide); declared with any N/A reasons at content.
- **FR12 — Coins nothing.** Binds frozen Buyer-leg contracts by pointer; never invokes the engine; on any gap → flag-and-halt (`[ESC-7-API]`), never invent.
